Architecture FastAPI
Version Starter — 100% Gratuit

MVP minimal sans dépendances payantes. Tous les services tiers coûteux sont désactivés mais prévus pour une activation future.
✓ FastAPI + Python ✓ PostgreSQL local ✓ Stockage local ✓ JWT simple ✓ WebSocket natif ✓ Notifs in-app Twilio SMS Firebase FCM Google OAuth Google Maps OCR ID card Celery
📌

Principe de cette version : Login par numéro + mot de passe hashé (bcrypt), vérification par email uniquement (lien de confirmation), stockage des fichiers dans un dossier local /uploads, notifications in-app uniquement (dans la base de données). Les services payants (Twilio, Firebase FCM, Google Maps, OCR) sont présents dans le code mais derrière un flag FEATURE_ENABLED=false — ils seront activés ultérieurement sans refactoring majeur.

04Stockage local des fichiers
📁

Dossier /uploads — servi par FastAPI via StaticFiles

Toutes les images sont stockées localement dans /app/uploads/. FastAPI les sert directement via StaticFiles montées sur /static. Le client accède aux images via https://api.ton-domaine.com/static/avatars/uuid.jpg. Lors du passage à Firebase Storage ou S3, seule la valeur en base (avatar_url) change — le reste du code REST ne bouge pas.
