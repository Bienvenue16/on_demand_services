02Couche Core — Infrastructure
CORE app/main.py

Point d'entrée de l'application. Crée l'instance FastAPI, connecte MongoDB au démarrage, monte tous les routeurs et sert les fichiers statiques.

  • Fonction lifespan() : connexion/déconnexion MongoDB Atlas via Beanie
  • Montage des 11 routeurs (include_router)
  • Middleware CORS (origines autorisées depuis .env)
  • Montage StaticFiles sur /static → dossier /uploads
  • Handler global des erreurs HTTP
CORE app/core/config.py

Classe Settings (pydantic-settings) qui lit et valide toutes les variables d'environnement du fichier .env.

  • MONGODB_URL, DB_NAME (depuis ton .env actuel)
  • SECRET_KEY, JWT_ALGORITHM, durées des tokens
  • SMTP_* pour l'envoi d'emails
  • Feature flags : PUSH_ENABLED, SMS_ENABLED...
  • Singleton get_settings() mis en cache
CORE app/core/database.py

Initialisation de Beanie ODM avec Motor (driver async pour MongoDB). Enregistre tous les Documents pour les collections.

  • Crée le client Motor à partir de MONGODB_URL
  • Appelle init_beanie(database, document_models=[...])
  • Liste tous les Documents : User, Proposal, Message, etc.
  • Fonction connect_db() appelée dans le lifespan
CORE app/core/security.py

Tout ce qui concerne l'authentification et la sécurité : JWT, hachage des mots de passe, dépendances FastAPI.

  • create_access_token() et create_refresh_token()
  • decode_token() avec gestion des erreurs
  • hash_password() et verify_password() via bcrypt
  • Dépendance get_current_user() injectée dans les routes
  • Dépendance get_admin_user() pour les routes admin
CORE app/core/email.py

Module d'envoi d'emails via SMTP (Gmail ou autre). Utilisé pour la vérification de compte et le reset de mot de passe.

  • send_verification_email(to, token)
  • send_reset_password_email(to, token)
  • Utilise smtplib + email.mime (bibliothèque standard Python)
  • Exécuté en arrière-plan via BackgroundTasks FastAPI
CORE app/core/exceptions.py

Exceptions HTTP personnalisées et handlers globaux pour des réponses d'erreur cohérentes dans toute l'API.

  • NotFoundException, ForbiddenException, ConflictException
  • Handler global FastAPI pour formater les erreurs en JSON uniforme
  • Handler pour RequestValidationError (Pydantic)
03Couche Models — Documents Beanie (MongoDB)
MODEL app/models/user.py

Collection users. Représente tout utilisateur de la plateforme (client, prestataire ou admin).

  • Champs : phone, email, password_hash, role (enum), is_verified
  • avatar_url (chemin local), full_name, created_at
  • email_token, reset_token, reset_token_expires
  • Index MongoDB sur email et phone (unique)
MODEL app/models/provider_profile.py

Collection provider_profiles. Données spécifiques aux prestataires, lié à un User.

  • user_id : référence vers User (PydanticObjectId)
  • specialties, service_zones, portfolio (liste URLs locales)
  • avg_rating (float), total_reviews, is_verified
  • bio, certificates (liste URLs)
MODEL app/models/service_request.py

Collection service_requests. Une demande de service publiée par un client.

  • user_id, category_id, title, description
  • lat, lng, address_text (géo manuelle)
  • budget, urgency (enum low/medium/high), status (enum)
  • photos (liste chemins locaux), created_at
MODEL app/models/proposal.py

Collection proposals. Une offre d'un prestataire en réponse à une demande.

  • request_id, provider_id
  • price, delay_hours, message
  • status : pending / accepted / declined
  • created_at
MODEL app/models/message.py

Collection messages. Historique persistant des messages WebSocket.

  • room_id (str), sender_id, content
  • media_url (image locale nullable), is_read
  • sent_at — Index sur room_id pour pagination rapide
MODEL app/models/review.py

Collection reviews. Évaluation laissée par un client après une prestation.

  • reviewer_id, provider_id, request_id
  • rating (1–5), comment, photos
  • Après insertion → met à jour avg_rating dans ProviderProfile
MODEL app/models/notification.py

Collection notifications. Notifications in-app stockées en base (version Starter).

  • recipient_id, type (enum : new_proposal, accepted...)
  • title, body, data (dict JSON libre)
  • is_read, created_at
MODEL app/models/category.py

Collection categories. Catégories hiérarchiques de services (avec sous-catégories).

  • name, description, icon
  • parent_id (auto-référence pour sous-catégories, nullable)
  • Peuplée via seed script au démarrage
04Couche Routers — Endpoints FastAPI
ROUTER app/routers/auth.py

Préfixe /auth. Tout ce qui concerne l'identité : inscription, connexion, tokens, vérification, reset.

  • POST /register — numéro + email + mot de passe → crée User
  • POST /login → retourne access + refresh token
  • POST /refresh → nouveau access token
  • POST /logout → invalide le token côté serveur
  • POST /verify-email → confirme le lien reçu par email
  • POST /forgot-password + /reset-password
ROUTER app/routers/users.py

Préfixe /users. Gestion des profils utilisateurs et des pages prestataires.

  • GET/PUT /me — profil connecté
  • DELETE /me — suppression RGPD
  • GET /{id} — profil public
  • GET /providers — liste filtrée (catégorie, zone)
  • GET /providers/{id}/portfolio
  • GET /me/stats — tableau de bord personnel
ROUTER app/routers/requests.py

Préfixe /requests. Publication et gestion des demandes de service.

  • POST / — publier une demande (auth requise)
  • GET / — liste avec filtres (catégorie, urgence, budget)
  • GET /nearby — filtrage par distance Haversine (lat/lng/radius)
  • GET /me, GET /{id}, PATCH /{id}
  • PATCH /{id}/status, DELETE /{id}
ROUTER app/routers/proposals.py

Préfixe /proposals. Cycle de vie des offres des prestataires.

  • POST / — soumettre une proposition
  • GET /me — mes propositions envoyées
  • GET /request/{req_id} — propositions reçues
  • PATCH /{id}/accept → notif in-app au prestataire
  • PATCH /{id}/decline, DELETE /{id}
ROUTER app/routers/messages.py

Préfixe /messages. Endpoints REST pour l'historique et le fallback de la messagerie.

  • GET /conversations — toutes mes conversations
  • GET /{room_id}/history — messages paginés
  • POST /{room_id} — envoyer sans WebSocket
  • PATCH /{room_id}/read — marquer comme lu
  • POST /{room_id}/media — envoyer une image
WS app/routers/websocket.py

Canal WebSocket temps réel. Authentification via token JWT en query param lors du handshake.

  • Endpoint : WS /ws/chat/{room_id}?token=...
  • Vérifie le JWT avant d'accepter la connexion
  • Délègue à ConnectionManager et handlers.py
  • Sauvegarde chaque message reçu en MongoDB
ROUTER app/routers/reviews.py

Préfixe /reviews. Soumission et lecture des évaluations.

  • POST / — soumettre un avis (1–5 étoiles)
  • GET /provider/{id} — avis d'un prestataire
  • GET /{id}, DELETE /{id}
ROUTER app/routers/notifications.py

Préfixe /notifications. Notifications in-app (stockées en MongoDB, version Starter).

  • GET / — mes notifications
  • GET /unread-count — badge non lus
  • PATCH /{id}/read, PATCH /read-all
ROUTER app/routers/categories.py + uploads.py

Catégories en lecture, upload local de fichiers.

  • GET /categories — liste hiérarchique
  • GET /categories/{id} — avec sous-catégories
  • POST /uploads/image — reçoit multipart, sauvegarde local, retourne URL
  • DELETE /uploads/{file_id} — supprime le fichier
ROUTER app/routers/admin.py

Préfixe /admin. Toutes les routes nécessitent le rôle admin (dépendance get_admin_user).

  • GET/PATCH /users, PATCH /users/{id}/ban
  • PATCH /providers/{id}/verify — validation manuelle KYC
  • GET/PATCH /reports — modération des signalements
  • GET /stats — tableau de bord global
  • POST/PUT/DELETE /categories — gestion des catégories
